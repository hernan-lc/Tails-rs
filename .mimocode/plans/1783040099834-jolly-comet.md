# Plan: Implement 8 Planned Modules

## Overview
Implement all 8 modules from the roadmap's "Planned (not yet implemented)" section under v0.5.0. Current NATIVE_TABLE_LEN = 412 (indices 0-411 used).

## Implementation Order
Modules are ordered by dependency and complexity (simplest first):

### Phase 1: Simple Built-in Modules (no new crates)

**1. `util` module** — `format`, `inspect`, `promisify`, `callbackify` (4 functions)
- Index range: 412-415
- File: `src/runtime_env/native_fns/util_fns.rs`
- `format`: printf-like string interpolation (`util.format("%s is %d", "age", 25)`)
- `inspect`: deep object inspection with configurable depth/colors
- `promisify(fn)`: returns a new function that wraps callback-based fn in a Promise
- `callbackify(fn)`: returns a new function that wraps promise-based fn with last-arg callback

**2. `events` expansion** — `prependListener`, `once`, `removeAllListeners`, `prependOnceListener`, `eventNames`, `getMaxListeners`, `setMaxListeners` (7 new functions)
- Index range: 416-422
- File: extend `src/runtime_env/native_fns/events_fns.rs`
- `once`: like `on` but auto-removes after first call (wrap callback to call `off` after invoke)
- `prependListener`: insert at front of listener array instead of push
- `prependOnceListener`: prepend + auto-remove
- `removeAllListeners(event?)`: remove all listeners, or all for a specific event
- `eventNames`: return array of events that have listeners
- `getMaxListeners`: return `_maxListeners` property (default 10)
- `setMaxListeners(n)`: set `_maxListeners` property
- Also add MaxListeners warning in `emit` when count exceeds `_maxListeners`
- Update `create_events_module` factory to include new prototype methods

**3. `timers` module** — `setImmediate`, `clearImmediate` (2 functions)
- Index range: 423-424
- File: `src/runtime_env/native_fns/timers_fns.rs`
- `setImmediate(callback, ...args)`: like setTimeout with 0 delay but uses microtask queue
- `clearImmediate(id)`: cancel a setImmediate
- Reuse existing `async_runtime.enqueue_macrotask` with 0 delay
- Register as module `timers` in native_loader (not as globals, matching Node.js)

**4. `querystring` module** — `parse`, `stringify`, `encode`, `decode` (4 functions)
- Index range: 425-428
- File: `src/runtime_env/native_fns/querystring_fns.rs`
- `parse(str, sep='&', eq='=')`: parse "a=1&b=2" to `{a:"1", b:"2"}`
- `stringify(obj, sep='&', eq='=')`: reverse of parse
- `encode(str)`: URL-encode (use `urlencoding::encode`)
- `decode(str)`: URL-decode (use `urlencoding::decode`)

### Phase 2: Complex Built-in Module

**5. `stream` module** — Readable/Writable/Transform/PassThrough streams (14 functions)
- Index range: 429-442
- File: `src/runtime_env/native_fns/stream_fns.rs`
- Readable: `readable_read`, `readable_pipe`, `readable_unpipe`, `readable_push`, `readable_destroy`
- Writable: `writable_write`, `writable_end`, `writable_destroy`, `writable_cork`, `writable_uncork`
- Transform: `transform_constructor`, `passthrough_constructor`
- Helpers: `stream_pipeline`, `stream_finished`
- Streams extend EventEmitter — constructor builds on events prototype
- Each stream has internal state: `_readableState`, `_writableState` stored as properties

### Phase 3: Feature-Gated Modules (require external crates)

**6. `zlib` module** — Compression/decompression (10 functions)
- Feature gate: `zlib`
- New crate dep: `flate2 = "1"`
- Index range: 443-452
- File: `src/runtime_env/native_fns/zlib_fns.rs`
- Sync: `gzip_sync`, `gunzip_sync`, `deflate_sync`, `inflate_sync`, `deflate_raw_sync`, `inflate_raw_sync`
- Async: `gzip`, `gunzip`, `deflate`, `inflate` (wrap sync in promise)
- Configurable compression level via options object
- Feature stubs in mod.rs when `zlib` feature is off

**7. `tls` module** — TLS/SSL support (5 functions)
- Feature gate: `tls`
- New crate dep: `native-tls = "0.2"`
- Index range: 453-457
- File: `src/runtime_env/native_fns/tls_fns.rs`
- `connect(options)`: create TLS connection (wraps native-tls connect)
- `createSecureContext(options)`: create TLS context with cert/key
- `TLSSocket`: wrapper around TCP socket with TLS
- `createServer(options)`: TLS server
- Feature stubs in mod.rs

**8. `dns` module** — DNS resolution (5 functions)
- Feature gate: `dns`
- New crate dep: `hickory-resolver = "0.24"`
- Index range: 458-462
- File: `src/runtime_env/native_fns/dns_fns.rs`
- `resolve(domain, rrtype?, callback)`: generic DNS resolution
- `lookup(hostname, options?, callback)`: address lookup (like getaddrinfo)
- `resolve4(domain, callback)`: resolve A records
- `resolve6(domain, callback)`: resolve AAAA records
- `resolveMx(domain, callback)`: resolve MX records
- Feature stubs in mod.rs
